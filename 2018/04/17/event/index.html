event

Posted on 2018-04-17
常见事件
鼠标事件: onclick, ondblclick, onmousedown, onmouseup, onmouseover, onmouseout, onmouseenter, onmouseleave
鼠标位置信息: event.clientX, event.clientY;
焦点事件: onfocus, onblur
select: 在表单元素中选中用户输入的内容
键盘事件 event.keyCode, event.ctrlKey, event.shiftKey, event.altKey

事件对象
chrom/ie, event是window身上的对象，直接用,FF中事件对象必需是第一个参数。

事件流
事件触发的流程 - 捕获阶段(从外向里) -> 目标阶段 -> 冒泡阶段(从里向外)。 - 所遇到的每个元素如果有相同的事件处理函数都会被触发。 - 低版本IE浏览器只绑定到冒泡阶段。 - 使用阻止冒泡方式可以让事件寻找停在阻止的位置。 - 目标阶段函数的执行顺序: 按照目标元素事件的书写顺序。

浏览器默认行为
并非JS添加的行为, 浏览器自带。

事件绑定的两种方法
对象.on事件名称＝函数; 注意：on只能给元素添加一个事件。如果说有多个on，那后面的会把前面的给覆盖了

移除事件: on事件 = null

阻止事件冒泡 event.cancelBubble = true

阻止浏览器默认行为 return false

addEventListener

参数1: 事件名称, 不带on的字符串

参数2: 事件绑定函数, 匿名或者命名都行, 函数中的this指向事件。

参数3: 布尔值, true表示捕获阶段发生事件, false表示在冒泡阶段发生。

对象.removeEventListener(事件名称,事件绑定函数,boolean);

不能移除掉匿名函数
只能移除同一阶段的绑定函数
阻止事件冒泡 event.stopPropagation()

阻止浏览器默认行为 event.preventDefault()

事件代理 target
事件源的元素event.target, DOM节点多的时候的常用优化是给父节点绑定事件, 然后用事件代理操作(非常重要)
